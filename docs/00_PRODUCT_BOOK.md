# SAM GUK CLUB TAEKWONDO
## PRODUCT BOOK

---

# Objetivo del proyecto

Desarrollar una plataforma web moderna para el Club Sam Guk Taekwondo que sirva como:

- Página oficial del club.
- Herramienta de captación de nuevos alumnos.
- Sistema de preinscripción online.
- Panel privado para la gestión de alumnos.
- Plataforma preparada para crecer en el futuro.

La experiencia debe transmitir profesionalidad, confianza y organización.

No queremos una web tradicional de gimnasio.

Queremos una marca moderna.

---

# Filosofía del proyecto

La web debe ser extremadamente sencilla de utilizar.

Debe poder utilizarla:

- un padre de 35 años
- una madre de 50 años
- un adolescente
- Leo como administrador

Todo debe sentirse claro.

Menos botones.

Menos texto.

Más calidad visual.

---

# Identidad visual

El diseño debe inspirarse en marcas premium.

Referencias:

- Apple
- Nike
- Notion
- Stripe

Nunca utilizar diseños antiguos de gimnasios.

El diseño debe sentirse limpio.

Mucho espacio en blanco.

Animaciones suaves.

Transiciones elegantes.

---

# Colores principales

Azul corporativo.

Rojo corporativo.

Blanco.

Grises muy claros.

Negro únicamente para texto.

---

# Fotografías

Toda la identidad visual utilizará fotografías reales del club.

Nunca utilizar fotografías de stock.

Las imágenes serán grandes.

Muy cinematográficas.

Con profundidad.

Con iluminación natural.

---

# Público objetivo

Padres que buscan un deporte para sus hijos.

Adolescentes.

Adultos.

Competidores.

Familias.

# Objetivos principales

1. Conseguir nuevas inscripciones.

# Alcance obligatorio de la V1

La primera versión deberá respetar las siguientes reglas de producto sin excepciones:

1. La inscripción pública crea únicamente una solicitud pendiente.
2. La inscripción pública nunca crea un alumno automáticamente.
3. Un alumno solo existe en el sistema después de que Leo revise y apruebe manualmente la solicitud.
4. El único acceso privado de la V1 pertenece a Leo y al personal expresamente autorizado.
5. La V1 no incluye portal privado para alumnos, familias o responsables.
6. Los pagos admitidos en la V1 son efectivo y, si Leo decide habilitarla, transferencia verificada manualmente.
7. La V1 no procesará pagos mediante pasarela online, tarjeta online ni Bizum automático.

Queda expresamente fuera del MVP:

- Multi-sede.
- Multi-gimnasio.
- Drag and drop.
- Analítica avanzada.
- Pasarela de pago.
- Roles complejos.
- Automatizaciones avanzadas.

Estos elementos podrán estudiarse en fases posteriores, pero no deberán generar pantallas, modelos, integraciones ni complejidad técnica dentro del MVP.

Estar preparado para crecer significa conservar datos consistentes, límites claros y una arquitectura mantenible. No significa implementar anticipadamente funcionalidades futuras.

# Módulos del proyecto

El proyecto estará dividido en seis grandes bloques.

1. Página pública.

2. Sistema de inscripción.

3. Panel privado.

4. Gestión de alumnos.

5. Documentación legal.

6. Preparado para futuras ampliaciones.

---

# Principios de desarrollo

Todo debe priorizar:

- rapidez
- simplicidad
- estética
- facilidad de mantenimiento
- escalabilidad

Cada decisión deberá responder a esta pregunta:

"¿Esto hace que Leo trabaje menos?"

Si la respuesta es sí, es una buena decisión.

Si la respuesta es no, no debe implementarse.
